# WePay V3 插件集成到主工程指南

## 问题与解决方案

### 1. 数据库统一：使用主工程的 PayDb

**问题**: V3 使用 PostgreSQL (pq/pqxx)，主工程使用 PayDb (SQLite)，会导致订单数据分裂。

**解决方案**: V3 直接复用主工程的 PayDb，不引入 pqxx。

#### 修改 OrderService.cpp

```cpp
// ❌ 旧代码（使用 PostgreSQL）
#include <pqxx/pqxx.h>
auto db = drogon::app().getDbClient();

// ✅ 新代码（使用主工程 PayDb）
#include "../../PayDb.h"

void OrderService::insertOrder(const Order& order) {
    // 使用主工程的 PayDb 单例
    auto& db = PayDb::instance();
    
    // 插入订单到 pay_order 表
    db.exec(
        "INSERT INTO pay_order (order_id, merchant_id, device_id, amount, pay_type, status, create_time) "
        "VALUES (?, ?, ?, ?, ?, ?, datetime('now'))",
        order.orderId,
        order.merchantId,
        order.deviceId,
        order.amount,
        order.payType,
        "pending"
    );
}

void OrderService::updateOrderStatus(const std::string& orderId, const std::string& status) {
    auto& db = PayDb::instance();
    
    db.exec(
        "UPDATE pay_order SET status = ?, update_time = datetime('now') WHERE order_id = ?",
        status,
        orderId
    );
}

Order OrderService::getOrder(const std::string& orderId) {
    auto& db = PayDb::instance();
    
    auto result = db.query(
        "SELECT order_id, merchant_id, device_id, amount, pay_type, status, create_time "
        "FROM pay_order WHERE order_id = ?",
        orderId
    );
    
    if (result.empty()) {
        throw std::runtime_error("Order not found");
    }
    
    Order order;
    order.orderId = result[0]["order_id"];
    order.merchantId = result[0]["merchant_id"];
    order.deviceId = result[0]["device_id"];
    order.amount = std::stod(result[0]["amount"]);
    order.payType = result[0]["pay_type"];
    order.status = result[0]["status"];
    order.createTime = result[0]["create_time"];
    
    return order;
}
```

#### 修改 CMakeLists.txt

```cmake
# ❌ 删除 PostgreSQL 依赖
# find_package(PostgreSQL REQUIRED)
# target_link_libraries(wepay_v3_plugin PUBLIC pq pqxx)

# ✅ 链接主工程的 PayDb
target_link_libraries(wepay_v3_plugin PUBLIC
    sqlite3  # PayDb 使用 SQLite
)
```

### 2. JSON 库统一：使用主工程的 jsoncpp

**问题**: 主工程用 jsoncpp (Json::Value)，V3 用 nlohmann::json，两套库会导致符号膨胀和接口转换问题。

**解决方案**: V3 改用 jsoncpp，或在边界做转换。

#### 方案 A：V3 全面改用 jsoncpp（推荐）

```cpp
// ❌ 旧代码（nlohmann::json）
#include <nlohmann/json.hpp>
nlohmann::json response;
response["code"] = 200;
response["message"] = "success";

// ✅ 新代码（jsoncpp）
#include <json/json.h>
Json::Value response;
response["code"] = 200;
response["message"] = "success";
```

#### 方案 B：边界转换（保留 nlohmann 内部使用）

```cpp
// 在 WepayV3Plugin 边界做转换
class JsonConverter {
public:
    // nlohmann::json -> Json::Value
    static Json::Value toJsonCpp(const nlohmann::json& nj) {
        Json::Value jv;
        std::string str = nj.dump();
        Json::CharReaderBuilder builder;
        std::istringstream iss(str);
        std::string errs;
        Json::parseFromStream(builder, iss, &jv, &errs);
        return jv;
    }
    
    // Json::Value -> nlohmann::json
    static nlohmann::json toNlohmann(const Json::Value& jv) {
        Json::StreamWriterBuilder builder;
        std::string str = Json::writeString(builder, jv);
        return nlohmann::json::parse(str);
    }
};

// 在 HTTP 接口边界使用
void DeviceController::handleRegister(
    const drogon::HttpRequestPtr& req,
    std::function<void(const drogon::HttpResponsePtr&)>&& callback) {
    
    // 接收 jsoncpp 格式
    auto jsonBody = req->getJsonObject();  // Json::Value*
    
    // 内部使用 nlohmann（如果需要）
    nlohmann::json internal = JsonConverter::toNlohmann(*jsonBody);
    
    // 处理逻辑...
    
    // 返回 jsoncpp 格式
    Json::Value response;
    response["code"] = 200;
    response["message"] = "success";
    
    auto resp = drogon::HttpResponse::newHttpJsonResponse(response);
    callback(resp);
}
```

#### 修改 CMakeLists.txt

```cmake
# ❌ 删除 nlohmann::json 依赖
# find_package(nlohmann_json REQUIRED)
# target_link_libraries(wepay_v3_plugin PUBLIC nlohmann_json::nlohmann_json)

# ✅ 使用 jsoncpp
find_package(jsoncpp REQUIRED)
target_link_libraries(wepay_v3_plugin PUBLIC jsoncpp)
```

### 3. Redis 连接池复用

**问题**: 每次请求都 `new Redis()`，浪费连接。

**解决方案**: 使用成员变量 `shared_ptr<Redis>`，复用连接池。

#### 修改 DeviceController.cpp

```cpp
// ❌ 旧代码（每次 new）
DeviceController::DeviceController() {
    auto redis = std::make_shared<sw::redis::Redis>("tcp://" + config.redis.host + ":" +
                                                     std::to_string(config.redis.port));
    deviceManager_ = std::make_shared<DeviceManager>(redis);
}

// ✅ 新代码（复用连接池）
class RedisPool {
public:
    static RedisPool& getInstance() {
        static RedisPool instance;
        return instance;
    }
    
    std::shared_ptr<sw::redis::Redis> getRedis() {
        if (!redis_) {
            auto& config = WepayV3Config::getInstance();
            
            sw::redis::ConnectionOptions opts;
            opts.host = config.redis.host;
            opts.port = config.redis.port;
            opts.password = config.redis.password;
            opts.db = config.redis.db;
            
            sw::redis::ConnectionPoolOptions pool_opts;
            pool_opts.size = 10;  // 连接池大小
            pool_opts.wait_timeout = std::chrono::milliseconds(100);
            
            redis_ = std::make_shared<sw::redis::Redis>(opts, pool_opts);
        }
        return redis_;
    }

private:
    RedisPool() = default;
    std::shared_ptr<sw::redis::Redis> redis_;
};

// 使用连接池
DeviceController::DeviceController() {
    auto redis = RedisPool::getInstance().getRedis();
    deviceManager_ = std::make_shared<DeviceManager>(redis);
}
```

#### 修改 OrderService.cpp

```cpp
// ❌ 旧代码
void OrderService::processOrder(const Order& order) {
    auto redis = sw::redis::Redis("tcp://...");  // 每次创建
    redis.set("order:" + order.orderId, order.toJson());
}

// ✅ 新代码
class OrderService {
public:
    OrderService(std::shared_ptr<sw::redis::Redis> redis)
        : redis_(redis) {}
    
    void processOrder(const Order& order) {
        redis_->set("order:" + order.orderId, order.toJson());
    }

private:
    std::shared_ptr<sw::redis::Redis> redis_;
};

// 在 WepayV3Plugin 中初始化
void WepayV3Plugin::init(const std::string& configFile) {
    auto redis = RedisPool::getInstance().getRedis();
    orderService_ = std::make_shared<OrderService>(redis);
}
```

```cpp
// ❌ 旧代码（读取 YAML）
#include <yaml-cpp/yaml.h>

bool WepayV3Config::loadFromFile(const std::string& configFile) {
    YAML::Node config = YAML::LoadFile(configFile);
    // ...
}

// ✅ 新代码（从主工程 config.json 读取）
#include <json/json.h>
#include <fstream>

bool WepayV3Config::loadFromMainConfig(const std::string& mainConfigFile) {
    std::ifstream ifs(mainConfigFile);
    if (!ifs.is_open()) {
        LOG_ERROR << "Failed to open config file: " << mainConfigFile;
        return false;
    }
    
    Json::Value root;
    Json::CharReaderBuilder builder;
    std::string errs;
    
    if (!Json::parseFromStream(builder, ifs, &root, &errs)) {
        LOG_ERROR << "Failed to parse config: " << errs;
        return false;
    }
    
    // 读取 wepay_v3 配置节
    if (!root.isMember("wepay_v3")) {
        LOG_ERROR << "Missing wepay_v3 config section";
        return false;
    }
    
    const Json::Value& v3Config = root["wepay_v3"];
    
    // Redis 配置
    redis.host = v3Config["redis"]["host"].asString();
    redis.port = v3Config["redis"]["port"].asInt();
    redis.password = v3Config["redis"]["password"].asString();
    redis.db = v3Config["redis"]["db"].asInt();
    
    // RocketMQ 配置
    rocketmq.namesrvAddr = v3Config["rocketmq"]["namesrvAddr"].asString();
    
    // Email 配置
    email.smtpHost = v3Config["email"]["smtpHost"].asString();
    email.smtpPort = v3Config["email"]["smtpPort"].asInt();
    email.username = v3Config["email"]["username"].asString();
    email.password = v3Config["email"]["password"].asString();
    email.fromEmail = v3Config["email"]["fromEmail"].asString();
    email.fromName = v3Config["email"]["fromName"].asString();
    email.useSsl = v3Config["email"]["useSsl"].asBool();
    
    // Security 配置
    security.hmacSecret = v3Config["security"]["hmacSecret"].asString();
    security.maxDevices = v3Config["security"]["maxDevices"].asInt();
    security.timestampWindow = v3Config["security"]["timestampWindow"].asInt();
    
    // 心跳超时
    heartbeatTimeout = v3Config["heartbeatTimeout"].asInt();
    
    return true;
}
```

### 5. RocketMQ 在 Windows/MinGW 下的兼容性

**问题**: rocketmq-client-cpp 在 Windows MinGW 下可能编译失败。

**解决方案**: 使用条件编译，Windows 下禁用 RocketMQ。

#### 修改 CMakeLists.txt

```cmake
# 检测操作系统
if(WIN32)
    option(ENABLE_ROCKETMQ "Enable RocketMQ support" OFF)
else()
    option(ENABLE_ROCKETMQ "Enable RocketMQ support" ON)
endif()

if(ENABLE_ROCKETMQ)
    find_package(RocketMQ REQUIRED)
    target_link_libraries(wepay_v3_plugin PUBLIC rocketmq::rocketmq)
    target_compile_definitions(wepay_v3_plugin PUBLIC ENABLE_ROCKETMQ)
endif()
```

#### 修改 WepayV3Plugin.cpp

```cpp
void WepayV3Plugin::init(const std::string& configFile) {
    // ... 其他初始化
    
#ifdef ENABLE_ROCKETMQ
    // 初始化 RocketMQ
    RocketMQProducerManager::getInstance().init(config.rocketmq.namesrvAddr);
    
    // 启动消费者
    orderPushConsumer_ = std::make_shared<OrderPushConsumer>(orderService_);
    orderPushConsumer_->start(config.rocketmq.namesrvAddr);
    
    LOG_INFO << "RocketMQ initialized";
#else
    LOG_WARN << "RocketMQ disabled (not supported on Windows)";
#endif
}

void WepayV3Plugin::shutdown() {
#ifdef ENABLE_ROCKETMQ
    // 关闭 RocketMQ
    if (orderPushConsumer_) orderPushConsumer_->shutdown();
    RocketMQProducerManager::getInstance().shutdown();
#endif
}
```

#### 修改 OrderService.cpp（降级处理）

```cpp
void OrderService::sendToRocketMQ(const Order& order) {
#ifdef ENABLE_ROCKETMQ
    // 使用 RocketMQ
    RocketMQProducerManager::getInstance().sendOrderPush(order.toJson());
#else
    // 降级：直接同步处理
    LOG_WARN << "RocketMQ disabled, processing order synchronously";
    processOrderSync(order);
#endif
}
```

### 7. 编译与运行

#### Windows MinGW 编译

```bash

mkdir build && cd build

# 配置（禁用 RocketMQ）
cmake .. -G "MinGW Makefiles" -DENABLE_ROCKETMQ=OFF

# 编译
mingw32-make -j4

# 运行
./your_main_app.exe
```

#### Linux 编译

```bash
# 创建构建目录
mkdir build && cd build

# 配置（启用 RocketMQ）
cmake .. -DENABLE_ROCKETMQ=ON

# 编译
make -j$(nproc)

# 运行
./your_main_app
```

## 总结

### 已解决的问题

1. ✅ **数据库统一**: V3 使用主工程的 PayDb (SQLite)，不引入 pqxx
2. ✅ **JSON 库统一**: V3 改用 jsoncpp，或在边界做转换
3. ✅ **Redis 连接池**: 使用 `shared_ptr<Redis>` 复用连接池
4. ✅ **配置文件统一**: 从主工程 `config.json` 读取 V3 配置
5. ✅ **RocketMQ 兼容**: Windows 下条件编译禁用 RocketMQ
6. ✅ **集成步骤**: 提供完整的集成指南

### 依赖库清单

**必需依赖**:
- Drogon (HTTP 框架)
- jsoncpp (JSON 解析)
- OpenSSL (加密)
- hiredis + redis++ (Redis 客户端)
- sqlite3 (数据库)
- Tesseract + Leptonica (OCR)
- libcurl (HTTP 客户端，邮件发送)

**可选依赖**:
- RocketMQ (消息队列，Linux 推荐)
- Prometheus C++ (监控指标，可选)
- OpenTelemetry (链路追踪，可选)
