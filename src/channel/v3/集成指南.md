# WePay V3 插件集成指南

## 集成方式

WePay V3 插件采用**源码直接引入**的方式集成到主项目中，无需编译成独立的库。

## 主程序集成示例

### 1. 在主程序中引入插件

**main.cpp:**
```cpp
#include <drogon/drogon.h>
#include "channel/v3/WepayV3Plugin.h"

using namespace wepay::v3;

int main() {
    // 初始化WePay V3插件
    auto& plugin = WepayV3Plugin::getInstance();
    if (!plugin.init("channel/v3/config.yaml")) {
        LOG_ERROR << "Failed to initialize WePay V3 Plugin";
        return -1;
    }

    LOG_INFO << "WePay V3 Plugin initialized successfully";

    // 配置Drogon
    drogon::app()
        .setLogPath("./logs")
        .setLogLevel(trantor::Logger::kInfo)
        .addListener("0.0.0.0", 8080)
        .setThreadNum(16)
        .setDocumentRoot("./public")
        .setUploadPath("./uploads")
        .setMaxConnectionNum(100000)
        .setIdleConnectionTimeout(60);

    // 注册信号处理
    signal(SIGINT, [](int) {
        LOG_INFO << "Shutting down...";
        WepayV3Plugin::getInstance().shutdown();
        drogon::app().quit();
    });

    LOG_INFO << "Server started on http://0.0.0.0:8080";

    // 运行Drogon
    drogon::app().run();

    return 0;
}
```

### 2. 主项目CMakeLists.txt

```cmake
cmake_minimum_required(VERSION 3.15)
project(your_project)

set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

# 查找依赖库
find_package(Drogon REQUIRED)
find_package(yaml-cpp REQUIRED)
find_package(OpenSSL REQUIRED)
find_package(hiredis REQUIRED)
find_package(redis++ REQUIRED)
find_package(RocketMQ REQUIRED)
find_package(Tesseract REQUIRED)
find_package(Leptonica REQUIRED)
find_package(nlohmann_json REQUIRED)
find_package(PostgreSQL REQUIRED)

# WePay V3 插件源文件
set(WEPAY_V3_SOURCES
    src/channel/v3/WepayV3Plugin.cpp
    src/channel/v3/WepayV3Config.cpp
    src/channel/v3/SecurityValidator.cpp
    src/channel/v3/DeviceManager.cpp
    src/channel/v3/HeartbeatController.cpp
    src/channel/v3/WebSocketController.cpp
    src/channel/v3/OrderService.cpp
    src/channel/v3/OcrService.cpp
    src/channel/v3/RocketMQManager.cpp
    src/channel/v3/DeviceController.cpp
)

# 你的主程序源文件
set(YOUR_SOURCES
    src/main.cpp
    # 你的其他源文件...
)

# 创建可执行文件
add_executable(your_app ${YOUR_SOURCES} ${WEPAY_V3_SOURCES})

# 链接库
target_link_libraries(your_app
    Drogon::Drogon
    yaml-cpp
    OpenSSL::SSL
    OpenSSL::Crypto
    hiredis::hiredis
    redis++::redis++
    rocketmq::rocketmq
    tesseract
    leptonica
    nlohmann_json::nlohmann_json
    pq
    pqxx
)

# 包含目录
target_include_directories(your_app PRIVATE
    ${CMAKE_CURRENT_SOURCE_DIR}/src
    ${YAML_CPP_INCLUDE_DIR}
    ${OPENSSL_INCLUDE_DIR}
    ${TESSERACT_INCLUDE_DIRS}
    ${PostgreSQL_INCLUDE_DIRS}
)
```

## 使用插件功能

### 获取服务实例

```cpp
#include "channel/v3/WepayV3Plugin.h"

// 获取插件实例
auto& plugin = wepay::v3::WepayV3Plugin::getInstance();

// 获取各个服务
auto emailService = plugin.getEmailService();
auto ocrService = plugin.getOcrService();
auto orderService = plugin.getOrderService();
auto deviceManager = plugin.getDeviceManager();

// 使用服务
emailService->sendPaySuccessEmail(emailData);
auto ocrResult = ocrService->recognizePaymentScreenshot(imagePath);
orderService->createOrder(orderInfo);
```

### 访问API接口

插件会自动注册以下HTTP/WebSocket端点到Drogon：

**HTTP接口：**
- `POST /api/wepay/v3/heart` - 心跳
- `POST /api/wepay/v3/push` - 订单推送确认
- `POST /api/wepay/v3/pending` - 待单查询
- `POST /api/wepay/v3/ocr` - OCR上传
- `POST /api/wepay/v3/device/register` - 设备注册
- `POST /api/wepay/v3/device/unbind` - 设备解绑
- `GET /api/wepay/v3/device/info` - 设备信息
- `GET /api/wepay/v3/callback/manual` - 手动回调
- `GET /health` - 健康检查
- `GET /metrics` - 系统指标

**WebSocket接口：**
- `WS /api/wepay/v3/ws` - 实时推单

```yaml

heartbeat_interval: 10
heartbeat_timeout: 180
order_timeout: 300

security:
  enable_hmac: true
  hmac_secret: "your-secret-key-here"
  timestamp_window: 300
  max_devices: 100

rocketmq:
  namesrv_addr: "127.0.0.1:9876"

  smtp_host: "smtp.gmail.com"
  smtp_port: 587

  password: "your-app-password"

  port: 6379

  endpoint: "http://127.0.0.1:9000"
  access_key: "minioadmin"
  secret_key: "minioadmin"
```

```bash
mkdir build && cd build

```

## 注意事项

1. **Drogon控制器自动注册**：所有Controller类会被Drogon自动发现和注册，无需手动注册
2. **配置文件路径**：确保config.yaml路径正确
3. **依赖库**：确保所有依赖库已安装
4. **PostgreSQL**：使用PostgreSQL而非MySQL
5. **RocketMQ**：确保RocketMQ服务已启动

## 常见问题

### Q: 如何禁用某些功能？
A: 在初始化后，可以选择性地不启动某些消费者或定时任务。

### Q: 如何自定义路由前缀？
A: 修改各个Controller中的路径定义。

### Q: 如何添加自定义中间件？
A: 使用Drogon的Filter机制添加全局或局部过滤器。

### Q: 如何扩展新功能？
A: 继承现有的Service类或创建新的Controller。
