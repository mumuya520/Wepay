## 1. Sentinel流量控制

### 功能概述
实现了完整的Sentinel流量控制机制，包括QPS限流、慢调用熔断、异常比例熔断。

### 核心组件

#### CircuitBreaker（熔断器）
- **三种状态**：CLOSED（关闭）、OPEN（打开）、HALF_OPEN（半开）
- **慢调用熔断**：当慢调用比例超过阈值时触发熔断
- **异常比例熔断**：当异常比例超过阈值时触发熔断
- **自动恢复**：熔断后等待一段时间自动进入半开状态测试恢复

#### RateLimiter（限流器）
- **令牌桶算法**：平滑限流，支持突发流量
- **预热机制**：系统启动时从低阈值逐渐增加到目标阈值
- **动态调整**：支持运行时调整限流阈值

### 使用示例

```cpp
// 初始化Sentinel
auto redis = std::make_shared<sw::redis::Redis>("tcp://127.0.0.1:6379");
SentinelFlowManager::getInstance().init(redis);

// 注册熔断规则
CircuitBreakerRule cbRule;
cbRule.resourceName = "order_push";
cbRule.threshold = 50;                  // 阈值
cbRule.timeWindow = 10;                 // 时间窗口10秒
cbRule.minRequestAmount = 10;           // 最小请求数
cbRule.slowRatioThreshold = 0.5;        // 慢调用比例50%
cbRule.maxSlowRequestRt = 1000;         // 慢调用定义：>1000ms
cbRule.errorRatioThreshold = 0.5;       // 异常比例50%
cbRule.retryTimeoutMs = 5000;           // 熔断5秒后重试
SentinelFlowManager::getInstance().registerCircuitBreakerRule(cbRule);

// 注册限流规则
RateLimitRule rlRule;
rlRule.resourceName = "order_push";
rlRule.threshold = 100;                 // QPS=100
rlRule.timeWindow = 1;                  // 1秒窗口
rlRule.warmUp = true;                   // 启用预热
rlRule.warmUpPeriodSec = 10;            // 预热10秒
SentinelFlowManager::getInstance().registerRateLimitRule(rlRule);

// 使用宏保护资源
void handleOrderPush() {
    SENTINEL_ENTRY("order_push");
    
    // 业务逻辑
    processOrder();
}

// 或者手动管理
void handleOrderPush() {
    auto entry = SentinelFlowManager::getInstance().entry("order_push");
    if (entry->isBlocked()) {
        return; // 被限流或熔断
    }
    
    try {
        processOrder();
        entry->recordSuccess();
    } catch (...) {
        entry->recordFailure();
        throw;
    }
}
```

### 熔断器状态转换

```
CLOSED (关闭) 
  ↓ 异常/慢调用比例超过阈值
OPEN (打开，拒绝所有请求)
  ↓ 等待retryTimeoutMs
HALF_OPEN (半开，允许1个请求测试)
  ↓ 成功 → CLOSED
  ↓ 失败 → OPEN
```

## 2. 灰度发布管理

### 功能概述
支持按设备ID、商户ID、百分比、白名单进行灰度发布，实现新功能的平滑上线。

### 灰度策略

#### 1. 设备ID灰度
指定特定设备ID列表，只有这些设备能使用新功能。

#### 2. 商户ID灰度
指定特定商户ID列表，只有这些商户的设备能使用新功能。

#### 3. 百分比灰度
根据哈希算法，按百分比随机分配用户到灰度组。

#### 4. 白名单灰度
指定白名单列表，灵活控制灰度范围。

### 使用示例

```cpp
// 初始化灰度管理器
auto redis = std::make_shared<sw::redis::Redis>("tcp://127.0.0.1:6379");
GrayscaleManager::getInstance().init(redis);

// 注册灰度规则 - 按设备ID
GrayscaleRule rule1;
rule1.featureName = "new_payment_flow";
rule1.strategy = GrayscaleStrategy::DEVICE_ID;
rule1.targetList = {"device001", "device002", "device003"};
rule1.enabled = true;
rule1.startTime = 0;
rule1.endTime = 0;
GrayscaleManager::getInstance().registerRule(rule1);

// 注册灰度规则 - 按百分比（10%用户）
GrayscaleRule rule2;
rule2.featureName = "new_ocr_engine";
rule2.strategy = GrayscaleStrategy::PERCENTAGE;
rule2.percentage = 10;
rule2.enabled = true;
rule2.startTime = std::time(nullptr);
rule2.endTime = std::time(nullptr) + 86400 * 7; // 7天后结束
GrayscaleManager::getInstance().registerRule(rule2);

// 检查是否命中灰度
if (GRAYSCALE_CHECK("new_payment_flow", deviceId, merchantId)) {
    // 使用新支付流程
    newPaymentFlow();
} else {
    // 使用旧支付流程
    oldPaymentFlow();
}

// 或使用宏自动返回
void handlePayment() {
    GRAYSCALE_ENABLED("new_payment_flow", deviceId, merchantId);
    
    // 只有命中灰度的用户才会执行到这里
    newPaymentFlow();
}

// 获取灰度统计
auto stats = GrayscaleManager::getInstance().getStats("new_payment_flow");
LOG_INFO << "Grayscale hit rate: " << stats.hitRate << "%";
```

### 灰度发布流程

```
1. 创建灰度规则（1%用户）
2. 观察1-2天，监控错误率
3. 逐步扩大灰度范围（5% → 10% → 50%）
4. 全量发布（100%）
5. 删除灰度规则
```

## 3. 设备分组管理

### 功能概述
支持按地区、商户、版本、自定义维度对设备进行分组管理，便于批量操作和统计分析。

### 分组类型

- **REGION**：按地区分组（华东、华南、华北等）
- **MERCHANT**：按商户分组
- **VERSION**：按App版本分组
- **CUSTOM**：自定义分组

### 使用示例

```cpp
// 初始化设备分组管理器
auto redis = std::make_shared<sw::redis::Redis>("tcp://127.0.0.1:6379");
DeviceGroupManager::getInstance().init(redis);

// 创建分组
DeviceGroup group;
group.groupId = "region_east";
group.groupName = "华东地区设备";
group.type = DeviceGroupType::REGION;
group.deviceIds = {"device001", "device002", "device003"};
group.metadata["region"] = "east";
group.metadata["city"] = "shanghai";
group.createTime = std::time(nullptr);
group.updateTime = std::time(nullptr);
DeviceGroupManager::getInstance().createGroup(group);

// 添加设备到分组
DeviceGroupManager::getInstance().addDeviceToGroup("region_east", "device004");

// 批量添加设备
std::vector<std::string> devices = {"device005", "device006", "device007"};
DeviceGroupManager::getInstance().batchAddDevices("region_east", devices);

// 获取设备所属分组
auto groups = DeviceGroupManager::getInstance().getDeviceGroups("device001");
for (const auto& groupId : groups) {
    LOG_INFO << "Device belongs to group: " << groupId;
}

// 获取分组内的所有设备
auto devices = DeviceGroupManager::getInstance().getGroupDevices("region_east");
LOG_INFO << "Group has " << devices.size() << " devices";

// 按类型获取分组
auto regionGroups = DeviceGroupManager::getInstance().getGroupsByType(DeviceGroupType::REGION);

// 获取分组统计
auto stats = DeviceGroupManager::getInstance().getGroupStats("region_east");
LOG_INFO << "Group: " << stats.groupName
         << " Total: " << stats.deviceCount
         << " Online: " << stats.onlineCount
         << " Offline: " << stats.offlineCount;
```

### 应用场景

1. **区域运营**：按地区分组，分析不同地区的支付情况
2. **版本管理**：按版本分组，灰度发布新版本功能
3. **商户管理**：按商户分组，统计商户设备状态
4. **批量操作**：对整个分组的设备进行批量配置更新

## 4. 告警推送集成

### 功能概述
集成钉钉、企业微信、短信三种告警渠道，支持多级告警和多渠道推送。

### 支持的告警类型

- **DEVICE_OFFLINE**：设备离线
- **ORDER_TIMEOUT**：订单超时
- **PAYMENT_FAILED**：支付失败
- **SECURITY_EVENT**：安全事件
- **SYSTEM_ERROR**：系统错误
- **PERFORMANCE**：性能告警

### 告警级别

- **INFO**：信息（ℹ️）
- **WARNING**：警告（⚠️）
- **ERROR**：错误（❌）
- **CRITICAL**：严重（🔥）

### 使用示例

```cpp
// 初始化告警管理器
nlohmann::json config;
config["dingtalk"] = nlohmann::json::array();
config["dingtalk"].push_back({
    {"name", "ops_group"},
    {"webhook", "https://oapi.dingtalk.com/robot/send?access_token=xxx"},
    {"secret", "SECxxx"}
});

config["wecom"] = nlohmann::json::array();
config["wecom"].push_back({
    {"name", "dev_group"},
    {"webhook", "https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxx"}
});

config["sms"] = {
    {"provider", "aliyun"},
    {"accessKeyId", "xxx"},
    {"accessKeySecret", "xxx"},
    {"signName", "WePay"},
    {"templateCode", "SMS_123456"}
};

AlertNotificationManager::getInstance().init(config);

// 发送告警
AlertMessage alert;
alert.type = AlertType::DEVICE_OFFLINE;
alert.level = AlertLevel::WARNING;
alert.title = "设备离线告警";
alert.content = "设备 device123 已离线超过5分钟";
alert.data["deviceId"] = "device123";
alert.data["offlineTime"] = "2024-01-23 10:30:00";
alert.timestamp = std::time(nullptr);
alert.channels = {AlertChannel::DINGTALK, AlertChannel::WECOM};

AlertNotificationManager::getInstance().sendAlert(alert);

// 发送到指定渠道
AlertNotificationManager::getInstance().sendToChannel(alert, AlertChannel::SMS);

// 获取告警统计
auto stats = AlertNotificationManager::getInstance().getStats();
LOG_INFO << "Total alerts: " << stats.totalAlerts
         << " Success: " << stats.successAlerts
         << " Failed: " << stats.failedAlerts;
```

### 钉钉消息格式

```markdown
## 设备离线告警

**级别**: ⚠️ 警告

**内容**: 设备 device123 已离线超过5分钟

**时间**: 1706000000

**详情**:
```json
{
  "deviceId": "device123",
  "offlineTime": "2024-01-23 10:30:00"
}
```
```

### 企业微信消息格式

```markdown
## 设备离线告警
> <font color="warning">⚠️ 警告</font>

**内容**: 设备 device123 已离线超过5分钟
**时间**: 1706000000
```

## 最佳实践

### 1. Sentinel使用建议
- 根据实际业务设置合理的阈值
- 慢调用阈值建议设置为P99响应时间的2倍
- 异常比例阈值建议设置为5-10%
- 熔断恢复时间建议设置为5-10秒

### 2. 灰度发布建议
- 从小范围开始（1-5%）
- 每次扩大灰度前观察1-2天
- 监控错误率、响应时间等关键指标
- 准备好快速回滚方案

### 3. 设备分组建议
- 合理规划分组维度
- 避免设备属于过多分组
- 定期清理无效分组
- 使用元数据存储额外信息

### 4. 告警推送建议
- 根据告警级别选择推送渠道
- INFO级别：仅记录日志
- WARNING级别：钉钉/企微
- ERROR/CRITICAL级别：钉钉+企微+短信
- 设置告警频率限制，避免告警风暴
