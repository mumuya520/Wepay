# WePay V3 新增功能说明

## 1. WebSocket连接管理（多实例部署支持）

### 功能概述
实现了基于Redis的WebSocket连接管理，支持多实例部署场景下的跨实例消息推送。

### 核心组件

#### WebSocketConnectionManager
- **本地连接映射**：使用`std::unordered_map`存储本实例的WebSocket连接
- **Redis持久化**：连接信息存储在Redis中，键格式为`ws:conn:{deviceId}`，TTL=300秒
- **服务器ID生成**：自动生成唯一ID（hostname-pid-timestamp-random）
- **跨实例消息推送**：通过Redis Pub/Sub实现消息转发

#### WebSocketMessageSubscriber
- **订阅Redis频道**：监听`ws:message`频道
- **消息路由**：根据`targetServerId`判断是否需要本实例处理
- **自动重连**：订阅失败时自动重试

### 使用示例

```cpp
// 在WepayV3Plugin中初始化
auto redis = std::make_shared<sw::redis::Redis>("tcp://127.0.0.1:6379");
wsConnectionManager_ = std::make_shared<WebSocketConnectionManager>(redis);
wsMessageSubscriber_ = std::make_shared<WebSocketMessageSubscriber>(redis, wsConnectionManager_.get());
wsMessageSubscriber_->start();

// 注册连接
wsConnectionManager_->registerConnection(deviceId, merchantId, conn, clientIp, userAgent);

// 推送消息（自动处理跨实例）
nlohmann::json message;
message["type"] = "ORDER_PUSH";
message["orderId"] = "123456";
wsConnectionManager_->pushMessageToDevice(deviceId, message);

// 更新心跳
wsConnectionManager_->updateHeartbeat(deviceId);

// 注销连接
wsConnectionManager_->unregisterConnection(deviceId);
```

### Redis数据结构

```json
// ws:conn:{deviceId}
{
  "deviceId": "device123",
  "merchantId": "merchant456",
  "serverId": "server-1234-5678",
  "connectTime": 1234567890,
  "lastHeartbeat": 1234567890,
  "clientIp": "192.168.1.100",
  "userAgent": "WePay-Android/1.0"
}
```

### 定时任务
- 每60秒清理过期连接（超过heartbeatTimeout未心跳）

## 2. OCR识别结果缓存

### 功能概述
使用SHA256哈希避免重复识别相同截图，大幅提升OCR性能。

### 核心功能

#### 图片哈希计算
- 使用OpenSSL EVP接口计算SHA256
- 支持从文件路径或内存数据计算
- 哈希值作为Redis缓存键

#### 缓存管理
- **缓存键格式**：`ocr:cache:{imageHash}`
- **默认TTL**：24小时（86400秒）
- **缓存内容**：识别结果（金额/支付类型/交易ID/置信度）
- **统计追踪**：记录缓存命中率

### 使用示例

```cpp
// 初始化OCR服务（带缓存）
auto redis = std::make_shared<sw::redis::Redis>("tcp://127.0.0.1:6379");
auto ocrCache = std::make_shared<OcrResultCache>(redis);
auto ocrService = std::make_shared<OcrService>(ocrCache);

// 识别截图（自动使用缓存）
auto result = ocrService->recognizePaymentScreenshot("/path/to/screenshot.jpg");

// 获取缓存统计
auto stats = ocrCache->getStats();
LOG_INFO << "Cache hit rate: " << stats.hitRate << "%";
```

### 缓存流程

```
1. 计算图片SHA256哈希
2. 查询Redis缓存
   - 命中：直接返回结果，记录hit
   - 未命中：执行OCR识别，缓存结果，记录miss
3. 返回识别结果
```

### Redis数据结构

```json
// ocr:cache:{imageHash}
{
  "imageHash": "abc123...",
  "amount": "100.50",
  "payType": "ALIPAY",
  "orderId": "",
  "transactionId": "2024012312345678",
  "recognizeTime": 1234567890,
  "confidence": 0.95,
  "success": true,
  "errorMessage": ""
}

// 统计
ocr:stats:hits = 1234
ocr:stats:misses = 567
```

## 3. 布隆过滤器（防缓存穿透）

### 功能概述
基于Redis BITMAP实现布隆过滤器，用于快速判断设备ID/订单ID是否存在，防止缓存穿透攻击。

### 核心特性

#### 算法实现
- **哈希算法**：MurmurHash3（高性能、低碰撞）
- **双哈希技巧**：使用两个哈希值组合生成多个哈希
- **自动优化**：根据预期元素数量和误判率计算最优参数

#### 参数计算
```cpp
// 位数组大小：m = -n * ln(p) / (ln(2)^2)
// 哈希函数数量：k = (m/n) * ln(2)
// 默认配置：n=1000000, p=0.01
// 结果：m≈9585059 bits (1.14MB), k=7
```

### 使用示例

```cpp
// 初始化布隆过滤器
auto redis = std::make_shared<sw::redis::Redis>("tcp://127.0.0.1:6379");
auto deviceBloomFilter = std::make_shared<BloomFilter>(
    redis, 
    "device",      // 过滤器名称
    1000000,       // 预期元素数量
    0.01           // 误判率1%
);

// 添加元素
deviceBloomFilter->add("device123");

// 检查元素
if (deviceBloomFilter->mightContain("device456")) {
    // 可能存在，需要查询数据库确认
} else {
    // 一定不存在，直接返回
}

// 批量添加
std::vector<std::string> deviceIds = {"device1", "device2", "device3"};
deviceBloomFilter->addBatch(deviceIds);

// 获取统计
auto stats = deviceBloomFilter->getStats();
LOG_INFO << "Bloom filter: " << stats.numBits << " bits, " 
         << stats.numHashFunctions << " hash functions";
```

### 应用场景

#### 1. 设备ID查询
```cpp
// 查询设备信息前先检查布隆过滤器
if (!deviceBloomFilter->mightContain(deviceId)) {
    return "Device not found";  // 快速返回，避免查询数据库
}
// 继续查询数据库
```

#### 2. 订单ID查询
```cpp
// 查询订单前先检查布隆过滤器
if (!orderBloomFilter->mightContain(orderId)) {
    return "Order not found";  // 避免缓存穿透
}
// 继续查询缓存或数据库
```

### Redis数据结构

```
bloom:device = BITMAP (9585059 bits)
bloom:order = BITMAP (95850590 bits for 10M orders)
```

## 4. 缓存预热（系统启动优化）

### 功能概述
系统启动时自动从PostgreSQL加载热点数据到Redis，避免冷启动时的缓存雪崩。

### 预热内容

#### 1. IP白名单
- 从`t_ip_whitelist`表加载所有启用的IP
- 存储到Redis SET：`ip:whitelist`

#### 2. 商户配置
- 从`t_merchant_config`表加载所有启用商户
- 存储到Redis STRING：`merchant:config:{merchantId}`
- 包含：回调URL、通知邮箱、HMAC密钥、最大设备数

#### 3. 在线设备
- 从`t_device_status`表加载最近5分钟心跳的设备
- 存储到Redis STRING：`device:status:{deviceId}`
- TTL=300秒

#### 4. 系统配置
- 从`t_system_config`表加载所有启用配置
- 存储到Redis STRING：`system:config:{configKey}`
- TTL=3600秒

#### 5. 布隆过滤器
- 加载所有设备ID到设备布隆过滤器
- 加载最近30天订单ID到订单布隆过滤器

### 使用示例

```cpp
// 在WepayV3Plugin初始化时调用
auto cacheWarmer = std::make_shared<CacheWarmer>(redis);
auto deviceBloomFilter = std::make_shared<BloomFilter>(redis, "device", 1000000, 0.01);
auto orderBloomFilter = std::make_shared<BloomFilter>(redis, "order", 10000000, 0.01);

// 执行预热
cacheWarmer->warmupAll();
cacheWarmer->warmupDeviceBloomFilter(deviceBloomFilter);
cacheWarmer->warmupOrderBloomFilter(orderBloomFilter);

// 获取预热进度
auto progress = cacheWarmer->getProgress();
LOG_INFO << "Cache warmup: " << progress.completedTasks << "/" << progress.totalTasks
         << " tasks, " << progress.failedTasks << " failed";
```

### 预热流程

```
1. 连接PostgreSQL数据库
2. 执行SQL查询获取数据
3. 批量写入Redis
4. 记录成功/失败统计
5. 返回预热进度
```

### 性能优化

- **批量操作**：使用Redis Pipeline减少网络往返
- **异步加载**：不阻塞主线程启动
- **错误容忍**：单个任务失败不影响其他任务
- **进度追踪**：实时监控预热状态

## 注意事项

1. **Redis内存**：布隆过滤器会占用一定内存，需要合理配置
2. **预热时间**：数据量大时预热可能需要较长时间
3. **误判处理**：布隆过滤器有1%误判率，需要后续数据库查询确认
4. **缓存一致性**：数据更新时需要同步更新缓存和布隆过滤器
5. **多实例部署**：确保所有实例都订阅了Redis Pub/Sub频道
