核心安全能力（V2 所有漏洞全部堵死 + V3 新增）
✅ 全链路 HMAC-SHA256 签名（无密钥无法伪造）
✅ RSA非对称签名（可选，设备私钥签名，服务端公钥验证）
✅ 时间戳 + nonce 双重防重放（重放包直接丢弃）
✅ 设备唯一身份认证（DeviceID + 绑定校验）
✅ IP 白名单（可配置）
✅ 金额强校验（不能改单）
✅ 订单幂等（不能重复入账）
✅ 多设备支持（不冲突）
✅ 心跳独立维护（在线状态精准）
✅ 严格协议格式（不兼容旧版漏洞协议）
✅ 日志审计（可追踪每一笔）


二、V3 协议规范（固定，不可伪造）

1. 公共请求字段（所有接口必须携带）

   deviceId    设备唯一ID（安装时生成并持久化）
   timestamp   Unix秒级时间戳（服务端校验±重放窗口内）
   nonce       随机字符串（每次请求不同，服务端去重）
   sign        HMAC-SHA256 签名

2. 设备端全部接口

   POST /api/wepay/v3/device/register   设备注册
   POST /api/wepay/v3/device/unbind     设备解绑
   POST /api/wepay/v3/heart             心跳上报（battery/network/appVersion）
   POST /api/wepay/v3/push             【主动确认】上报 orderId + status(PAID/FAILED)
   POST /api/wepay/v3/notify           【通知栏】上报 payType + amount [+ remark]
   POST /api/wepay/v3/pending           查询待推送订单（WS断线兜底）
   POST /api/wepay/v3/ocr              上传支付截图（multipart/form-data）
   GET  /api/wepay/v3/callback/manual  手动回调（邮件链接，含 token 签名）
   WS   /api/wepay/v3/ws              WebSocket 双向推单通道

3. 签名规则

   sign = HMAC-SHA256(
       "deviceId=xxx&nonce=xxx&timestamp=xxx&key=secret",
       secret
   )
   secret = 系统配置中的 hmac_secret（或各商户独立密钥）

4. /notify 通知栏接口特别说明

   请求：{ deviceId, payType, amount, remark(可选), timestamp, nonce, sign }
   响应（匹配成功）：{ matched:true, orderId, merchantOrderId, amount, payTime }
   响应（无匹配）：  { matched:false, reason:"no pending order matches..." }
   匹配逻辑：查询设备所有绑定商户 → 按金额+payType找最旧PENDING订单 → FIFO


V3 新增能力
🆕 通知栏监听模式（/notify）：设备无需持有 orderId，服务端按金额反查匹配
🆕 WebSocket双向推送（服务端主动推单，取代设备轮询）
🆕 WS多实例支持（Redis Pub/Sub跨实例转发 + 单实例本地映射降级）
🆕 截图OCR上传（/api/wepay/v3/ocr，Tesseract/PaddleOCR识别金额）
🆕 OCR结果缓存（SHA256去重，相同截图不重复识别，TTL=24h）
🆕 支付截图存MinIO（留存凭证，可追溯）
🆕 多商户绑定（一台设备监听多个商户，v3_device_merchant N:M映射）
🆕 RSA非对称签名（比HMAC更安全，设备持有私钥）
🆕 设备实时状态上报（电量/网络类型/App版本/屏幕分辨率）
🆕 限流熔断（Sentinel QPS/慢调用/异常比例熔断）
🆕 布隆过滤器（Redis BITMAP，防DeviceID/OrderID缓存穿透）
🆕 缓存预热（启动时加载IP白名单/商户配置/在线设备/系统配置）
🆕 灰度发布（GrayscaleManager，按设备/商户/百分比，Redis不可用时全量放开）
🆕 异步任务队列（RocketMQ 7个Topic，事务消息+延迟消息+顺序消息）
🆕 邮件通知与手动回调（SMTP邮件+带签名手动回调链接）
🆕 告警推送（AlertNotification，钉钉/企微Webhook + 邮件，防抖30min）
🆕 管理后台完整API（17个接口，统计/设备/订单/配置/白名单/审计/告警）
🆕 前端管理页面（7个独立页面 + 插件控制台监控Tab）
🆕 收款码管理（上传图片自动识别pay_url，支持微信/支付宝/QQ）
🆕 Redis不可用降级（WS单实例模式 / 灰度全量放开 / 布隆过滤跳过）


三、核心业务流程详解
================================================================================

【主动推单模式】完整流程：
─────────────────────────────────────────────────────────
 商户下单请求
      │
      ▼
 WePay 网关创建 v3_order（状态 PENDING）
      │
      ├─ WS在线 ──▶ WebSocketManager.pushOrderToDevice()
      │               Redis Pub/Sub 跨实例广播
      │               设备收到 {orderId, amount, payType, expireAt}
      │
      └─ WS离线 ──▶ 写 RocketMQ order-push-topic（设备轮询 /pending 兜底）
      │
      ▼（顾客支付后）
 设备收到微信/支付宝通知 → POST /api/wepay/v3/push
      { deviceId, orderId, status:PAID, timestamp, nonce, sign }
      │
      ▼
 服务端精确匹配 orderId → 金额强校验 → 幂等检查
      │
      ▼
 更新 v3_order.status=PAID → 写 v3_order_status_log
      │
      ▼
 触发商户回调 + 发送邮件通知 + 告警（如有异常）

【通知栏监听模式】完整流程：
─────────────────────────────────────────────────────────
 顾客扫固定收款码（v3_qrcode 表中的码）直接支付
      │
      ▼
 微信/支付宝 App 推通知到设备通知栏
 设备 AccessibilityService 捕获通知内容（金额、支付方式）
      │
      ▼
 POST /api/wepay/v3/notify
      { deviceId, payType:WECHAT, amount:100.00, remark, timestamp, nonce, sign }
      │
      ▼
 HMAC 验签 → 查询设备绑定的所有商户
      │
      ▼
 遍历商户 → matchOrderByAmount(merchantId, 100.00, WECHAT)
      SQL: v3_order WHERE status=PENDING AND ABS(amount-100.00)<0.001
           AND pay_type=WECHAT AND expire_at>now ORDER BY created_at ASC LIMIT 1
      │
      ├─ 匹配成功 ──▶ UPDATE v3_order SET status=PAID
      │               写 v3_order_status_log（来源：通知栏监听）
      │               触发商户回调
      │               返回 { matched:true, orderId, amount, payTime }
      │
      └─ 无匹配   ──▶ LOG_INFO 记录（供人工对账）
                      返回 { matched:false, reason }

【心跳 & 离线检测】：
─────────────────────────────────────────────────────────
 设备每10s POST /api/wepay/v3/heart
      { deviceId, battery, network, appVersion, timestamp, nonce, sign }
      │
      ▼
 更新 v3_device.last_heartbeat + Redis device:{deviceId} HASH
      │
      ▼
 定时任务每30s扫描 v3_device WHERE last_heartbeat < now-180s
      │
      ▼
 标记 online=false → AlertNotification 触发设备离线告警
      推送渠道：钉钉Webhook / 企微Webhook / SMTP邮件（按 v3_system_config 配置）


四、容灾与高可用设计
┌──────────────────────────────────────────────────────────────────────────────┐
│  1. 服务高可用                                                                │
│     ├─ 多实例部署：Nginx负载均衡，至少2个实例                                │
│     ├─ 健康检查：自动摘除故障节点                                            │
│     └─ 无状态设计：WebSocket连接信息存Redis，支持实例间漂移                  │
│                                                                               │
│  2. 数据高可用                                                                │
│     ├─ MySQL主从复制：读写分离，从库故障自动切换                             │
│     ├─ Redis哨兵/集群：自动故障转移                                          │
│     ├─ RocketMQ高可用：NameServer集群+Broker主从同步，消息持久化             │
│     └─ MinIO分布式部署：多副本存储，数据冗余                                 │
│                                                                               │
│  3. 降级策略                                                                  │
│     ├─ WebSocket降级：WS断线时自动切换到HTTP轮询pending接口                  │
│     ├─ OCR降级：识别失败时人工审核，或使用备用OCR引擎                        │
│     ├─ RocketMQ降级：消息队列故障时同步处理，记录失败日志人工补偿            │
│     ├─ 限流降级：过载时拒绝非核心接口（如统计查询），保障核心支付流程        │
│     └─ 缓存降级：Redis故障时直接查数据库（性能下降但不影响功能）             │
│                                                                               │
│  4. 灾难恢复                                                                  │
│     ├─ 数据库定时备份：每日全量+每小时增量，保留30天                         │
│     ├─ RocketMQ消息备份：重要消息持久化到MySQL，可追溯补偿                   │
│     ├─ 截图文件备份：MinIO跨区域复制，防止单点故障                           │
│     ├─ 配置备份：系统配置版本化管理（Git）                                   │
│     └─ 应急预案：故障演练、回滚流程、紧急联系人                              │
└──────────────────────────────────────────────────────────────────────────────┘


六、性能优化策略

七、安全加固措施
┌──────────────────────────────────────────────────────────────────────────────┐
│  1. 传输安全                                                                  │
│     ├─ HTTPS强制：生产环境强制TLS 1.2+，禁用弱加密套件                       │
│     ├─ WSS加密：WebSocket使用wss://协议，防止中间人攻击                      │
│     └─ 证书管理：Let's Encrypt自动续期，证书过期监控                         │
│                                                                               │
│  2. 访问控制                                                                  │
│     ├─ IP白名单：生产环境严格限制IP，支持动态添加/删除                       │
│     ├─ 设备黑名单：异常设备自动拉黑，支持手动解封                            │
│     ├─ 商户隔离：不同商户数据完全隔离，防止越权访问                          │
│     └─ 管理后台：RBAC权限控制，操作日志审计                                  │
│                                                                               │
│  3. 数据安全                                                                  │
│     ├─ 敏感数据加密：密钥、RSA私钥使用AES256加密存储                         │
│     ├─ 数据脱敏：日志中自动脱敏手机号、金额等敏感信息                        │

│     └─ 异常行为检测：单设备请求频率异常自动告警                              │
